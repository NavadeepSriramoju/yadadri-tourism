## 🏗️ Architecture

### Service Layer Pattern
- **DarshanTimingService**: Manage temple visit timings
- **BookingService**: Handle visitor bookings
- **UserService**: Authentication & authorization

### DTO Pattern
All API requests/responses use DTOs for validation and security:
- `DarshanTimingDto`
- `BookingDto`
- `BookingStatsDto`
- `ErrorResponse`

### Exception Handling
Global exception handler provides consistent error responses:
- 400: Validation errors with field details
- 401: Unauthorized access
- 404: Resource not found
- 500: Server errors with stack trace

## 🛠️ API Endpoints

### Public Endpoints
```
GET  /api/darshan-timings     - List all active timings
GET  /api/darshan-timings/:id - Get timing details
POST /api/bookings            - Create a booking
GET  /health                  - Health check
```

### Admin Endpoints (Protected)
```
POST   /api/darshan-timings      - Create timing
PUT    /api/darshan-timings/:id  - Update timing
DELETE /api/darshan-timings/:id  - Delete timing
GET    /api/bookings             - View all bookings
PATCH  /api/bookings/:id/status  - Update booking status
```
